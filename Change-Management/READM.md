Change Management

📌 Description

Change Management is used to control and manage changes in the IT environment with minimal risk. This project demonstrates how different types of changes are created, approved, and implemented in ServiceNow.

🎯 Objectives

- Manage changes in a structured way
- Reduce risk during deployments
- Ensure proper approvals before implementation

🧩 Change Types

- Normal Change – Requires approval process
- Standard Change – Pre-approved, no need of repeated approvals
- Emergency Change – High priority, handled quickly

⚙️ Features Implemented

- Change Request creation
- Risk & Impact fields usage
- Approval workflows (Manager / CAB)
- Change lifecycle tracking
- Work Notes and State updates

🔄 Workflow

New → Assess → Authorize → Implement → Review → Closed

🛠️ Tools Used

- ServiceNow
